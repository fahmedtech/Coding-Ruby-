IT 211 -- Project 2
Write Pseudocode and Rock-Paper-Scissors Game

Part A (50 points): Write Pseudocode for an Algorithm that Occurs in Everyday Life

  • Write pseudocode for an algorithm that occurs in everyday life.
     
  • Your pseudocode should have at least one if statement and at least one while loop.
     
  • Look at the Baking Bread, Washing Clothes, and Sorting Mail algorithms in the Structured Programming Details document.
     
  • Some examples of algorithms that occur in everyday life:
    1. Deciding which golf club to choose when hitting a ball.
    2. Deciding what to buy at the grocery store.
    3. Detailed steps for calling a friend. (Include what happens when line is busy or there is no answer.)
    4. Deciding which clothes to wear when getting dressed in the morning.
    5. Deciding who to ask out on a date.

  • Grading breakdown: Correctness: 75%; Originality/Creativity: 15%; Indentation: 10%.

Part B (100 points): Implement a Rock-Paper-Scissors Game

  • Write a Ruby script that allows a human player to play the Rock-Paper-Scissors game against the computer.
     
  • To play the game, the human player chooses one of "rock," "paper," or "scissors." The computer also chooses, randomly with equal probability, one of these three plays.
     
  • These phases determine the winner for each combination: "rock crushes scissors", "scissors cuts paper," and "paper covers rock."
     
  • In this diagram, each arrow points from the winner to the user.
     
  • If both the human and the computer choose the same play, there is a tie.
     
  • Here is a sample script output. The human player's choices are shown in red.
      Enter human play (rock, paper, scissors): paper
Computer plays rock.
Paper covers rock.
Human wins.
Score: Human 1, Computer 0.

Enter human play (rock, paper, scissors): scissors
Computer plays scissors.
Tie.
Score: Human 1.5, Computer 0.5.

Enter human play (rock, paper, scissors): scissors
Computer plays rock.
Rock crushes scissors.
Computer wins.
Score: Human 1.5, Computer 1.5.
  • You can base your script on this pseudocode:
      set human score to 0.0
set computer score to 0.0

while true

  input human play with prompt
  choose random number from 0, 1, or 2
  
  if random number is 0
    set computer play to rock
  elsif random number is 1
    set computer play to paper
  elseif random number is 2
    set computer play to scissors
  end
  
  if human plays rock and computer plays rock
    Result is tie.
    Add 0.5 to each score.
  elsif human plays rock and computer plays paper
    Result is computer wins.
    Add 1 to computer score.
  elsif human plays rock and computer plays scissors
    Result is human wins.
    Add 1 to human score.
  elsif human plays paper and computer plays rock
    Result is human wins.
    Add 1 to human score.
  elsif human plays paper and computer plays paper
    Result is tie.
    Add 0.5 to each score.
  elsif human plays paper and computer plays scissors
    Result is computer wins.
    Add 1 to computer score.
  elsif human plays scissors and computer plays rock
    Result is computer wins.
    Add 1 to computer score.
  elsif human plays scissors and computer plays paper
    Result is human wins.
    Add 1 to human score.
  elsif human plays scissors and computer plays scissors
    Result is tie.
    Add 0.5 to each score.
  end
  display human score and computer score

end
  • Use Control-C to interrupt and terminate your script when you are finished playing the game.

Whereas the original game has 9 (3 × 3) possible move combinations, the enhanced game has 25 (5 × 5) move combinations.
